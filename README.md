# Project Euler Notebooks

Notebook-based solutions and notes for selected [Project Euler](https://projecteuler.net/) problems.

The focus of this repository is not just getting final answers. Each notebook is written as a small mathematical programming exercise, usually including:

* a brute force approach,
* a more efficient approach,
* complexity analysis,
* Python implementation,
* tests or consistency checks,
* short notes on what I learned.

This repository is mainly intended to document my progress in mathematical problem solving, Python, algorithms, and explanatory writing.

## Disclaimer

Project Euler problems belong to Project Euler. I do not claim ownership of the original problem statements.

To avoid turning this repository into a direct answer dump, I generally link to the original problem rather than reproducing full problem text. The explanations, code, and notes are my own.

I also avoid publishing solutions to later Project Euler problems where public solution-sharing would go against the spirit of the site.

## Why notebooks?

I use notebooks because they let me combine mathematical reasoning, code, output, and commentary in one place. This is useful for Project Euler because many problems are not just coding exercises; they are also small lessons in number theory, combinatorics, search, optimisation, or algorithmic thinking.

This notebook contains my own working notes and Python solution for a Project Euler problem. The original problem is available at the linked Project Euler page. I avoid reproducing the full problem statement where possible.
